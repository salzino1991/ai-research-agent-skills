# Deep Research Report Template

Use this template when the user asks for deep research, 공략법, 자료수집, comparison, or a source-backed recommendation.

## 1. Executive summary

State the answer in 3-6 sentences.

Include:

- the main conclusion
- the highest-impact recommendation
- major caveat or uncertainty
- who this answer is for

## 2. Best answer / 공략 핵심

Give the core strategy in a compact form.

Recommended format:

```text
Best path: <recommended path>
Why: <main reason>
Avoid: <main trap>
Verify: <how to confirm this applies>
```

## 3. Step-by-step action plan

Provide an ordered plan the user can execute.

Each step should include:

- action
- reason
- verification method
- risk or failure condition

## 5. Source conflicts and uncertainty

List any disagreements between sources.

For each conflict:

- what sources disagree about
- which side is more reliable
- what condition changes the answer
- what should be checked next

## 6. Common mistakes / traps

List pitfalls that would cause the user to make a poor decision.

Examples:

- relying on outdated advice
- optimizing the wrong metric
- ignoring version, region, or platform differences
- trusting anecdotal community consensus over official rules
- missing hidden costs or risks

## 7. Final recommendation

Give a clear recommendation.

Use one of these confidence labels:

- **High confidence**: strong, current, primary evidence and little disagreement.
- **Medium confidence**: good evidence but some context dependence or moderate disagreement.
- **Low confidence**: weak evidence, high uncertainty, or fast-changing topic.

## 8. Follow-up research checklist

List what should be checked next if the user wants a more exhaustive answer.

Keep it practical:

- exact source to verify
- exact search query to run
- exact metric to compare
- exact test or experiment to perform
